# Style catalogue

Choose from this table using level first, formality second, and vibe third. If the user names a style, honor it. Otherwise default to a Balanced style that matches the content.

After choosing, open only the linked style file. Each style is adapted to JSON Canvas color and layout capabilities; it does not promise custom borders, typefaces, shadows, or CSS.

Source: [beautiful-feishu-whiteboard](https://github.com/zarazhangrui/beautiful-feishu-whiteboard), commit `6989843b355ac92ebbd4f66166189a001e61e9b5`, MIT License.

- **Restrained**: quiet, serious, editorial, or institutional.
- **Balanced**: a clear accent on a calm structure; safest for general work.
- **Bold**: playful, energetic, or attention-grabbing; use lower information density.

## Selection rules

1. Honor a style the user already named.
2. For a new board or full restyle without a named style, shortlist exactly three relevant styles. Put the recommendation first and show each style's formality, vibe, and palette in one compact line.
3. Ask the user to choose one or delegate the choice. Do not open a style file or build the board until the user chooses, unless the user already delegated.
4. When the user delegates, choose the best fit from content, audience, formality, and desired energy; state the choice and reason in one line.
5. For a targeted layout or connector repair, preserve the current style unless the user asks to change it.
6. Use no more than three active accent colors in one board.
7. Keep content and layout unchanged when the user asks only to switch styles.
